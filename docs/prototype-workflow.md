# Prototype Workflow

Prototype work should be small, testable, and easy to revert.

## Process

1. Start from `agent/pointer.md`.
2. Read the selected prompt.
3. Read `agent/workflows/prototype-workflow.md`.
4. Build the smallest proof that answers the question.
5. Prefer a route, isolated surface, or small data change over a broad rewrite.
6. Validate with build, route, browser, or screenshot evidence when possible.
7. Record detailed proof in `agent/reports/` if needed.
8. Update `output.md` with one short summary.

## Promotion Rule

A prototype is not promoted just because it exists.

Promote it only after the proof is clear and the next owner is known.

## Runtime Rule

If the prototype needs reusable runtime behavior, review `agent/dependencies.md` before placing it in Lost Pages.
