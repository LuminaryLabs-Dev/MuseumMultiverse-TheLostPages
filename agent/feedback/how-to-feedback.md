# How To Handle Feedback

Status: active

## Intake

When the user gives feedback, decide if it is durable enough to write into the repo.

Durable feedback is explicit, repeated, or important for future runs.

## Process

1. Add current useful feedback to `active-feedback.md`.
2. Read active feedback before editing.
3. Convert feedback into a prompt only after reading `agent/pointer.md`.
4. When feedback is addressed, move it to `processed-feedback.md`.
5. If feedback changes the operating rules, update `agent/memory.md`.
6. If feedback changes agent files, update `agent/change-log.md`.

## Style

Keep raw user wording when it helps preserve intent. Also add a short normalized meaning.
