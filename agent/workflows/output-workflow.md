# Output Workflow

Status: active

## Use When

Use this workflow for deploy chat, output.md, and output-rules.md work.

## Steps

1. Read output-rules.md.
2. Update output.md with the shortest useful message.
3. Avoid repo metadata unless requested.
4. Keep old history out of output.md.
5. Confirm the deploy workflow still reads output.md.
6. Update the pointer and run log.

## Acceptance

The next deploy chat message should be short, current, and user-facing.
