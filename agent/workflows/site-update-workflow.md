# Site Update Workflow

Status: active

## Use When

Use this workflow for normal Lost Pages source changes.

## Steps

1. Inspect the related source files.
2. Make one bounded change.
3. Run the closest build or static check when possible.
4. Update output.md with a short result.
5. Update pointer.md and run-log.md.

## Acceptance

The app should still build or the blocker should be clearly recorded.
