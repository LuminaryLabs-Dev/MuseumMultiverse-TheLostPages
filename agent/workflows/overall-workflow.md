# Overall Workflow

Status: active

Use this workflow when a run is broad, cross-cutting, or about project direction.

## Order

1. Start from agent state, not source files.
2. Read pointer, goal, dependencies, feedback, and memory.
3. Resolve the selected prompt.
4. Decide the smallest useful change.
5. Inspect source only after the run target is clear.
6. Validate with the closest available check.
7. Update output.md, run-log.md, and pointer.md.

## Self Learning Loop

Feedback becomes active feedback.

Active feedback becomes prompts or edits.

Run results become run log entries.

Durable rules become memory.

Agent system changes become change log entries.

The next task becomes the pointer.
